## Design Context

### Users
TIA Studio is for developers and operator-style power users who run local AI workflows. They use the app to start focused chats with ACP-backed agents, connect assistants to workspaces and channels, and only occasionally dive into deeper TIA-native configuration.

### Brand Personality
Calm, capable, technical. The interface should feel confident and uncluttered, with the product acting like a reliable workstation instead of a flashy consumer chat app.

### Aesthetic Direction
Adopt an AionUI-inspired desktop workspace: restrained dark surfaces, a simple three-pane chat shell, and an empty state that foregrounds agent choice and intent. The main page should be ACP-first and lightweight, while advanced TIA harness setup lives in Settings as a secondary flow.

### Design Principles
1. Chat first: the default screen should open into a focused conversation workspace, not a configuration dashboard.
2. ACP first: Claude, Codex, Gemini, Qwen Code, and OpenClaw should feel like the primary starting points.
3. Advanced stays secondary: TIA-native harnesses, provider wiring, and deeper operations live in Settings.
4. Context stays visible: the right rail should always explain the current session, workspace, and readiness without taking over the main canvas.
5. Minimal but warm: remove visual clutter, but keep enough hierarchy and polish that the app still feels intentional and premium.
