# v0.3.0

## Highlights

- Added `agent-browser` as a managed runtime option in Runtime Setup, including install-from-release support and custom local binary selection.
- Updated coding-agent runtime inference so `npx`-style skill install requests can resolve to the managed `bunx` runtime when Bun is configured.
- Expanded the managed runtime sandbox/writable roots so app-managed runtime binaries like `bunx` and `agent-browser` are available to the workspace safely.

## Platform And Release Improvements

- Updated the CI and release workflows to use Node 22 so dependency installation stays aligned with the runtime stack across GitHub Actions.
- Kept the managed runtime tests and setup flow aligned with the new `agent-browser` option so the runtime settings page and release artifacts stay in sync.

## Notes

- `v0.3.0` is the next public minor release after `v0.2.0`, focused on managed runtime onboarding and the CI updates needed to ship it reliably.
